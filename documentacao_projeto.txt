# Diagrama MVC (Model-View-Controller)

O projeto segue um padrão MVC simplificado em PHP puro:

+-------------------------------------------------------------+
|                                                             |
|                         USUÁRIO                             |
|                            |                                |
+----------------------------|--------------------------------+
                             |
                             v
+-------------------------------------------------------------+
|                      VIEW (Visão)                           |
|  - public/ (Formulários)                                    |
|  - tables/ (Listagens)                                      |
|  - components/ (Header, Nav)                                |
|  - css/ & js/                                               |
+----------------------------|--------------------------------+
                             |
                   (Ação / Requisição POST/GET)
                             |
                             v
+-------------------------------------------------------------+
|                   CONTROLLER (Controle)                     |
|  - api/ (Processamento de Formulários)                      |
|  - auth/ (Verificação de Login)                             |
|  - tables/api/ (Lógica de dados para tabelas)               |
+----------------------------|--------------------------------+
                             |
                    (Consulta / Escrita)
                             |
                             v
+-------------------------------------------------------------+
|                     MODEL (Modelo)                          |
|  - config/conexao.php (Conexão MySQL)                       |
|  - Banco de Dados (MySQL / database.sql)                    |
+-------------------------------------------------------------+

## Fluxo de Dados:
1. O usuário interage com a **VIEW** (ex: preenche um formulário em `form_ocorrencia.php`).
2. A **VIEW** envia os dados para o **CONTROLLER** (ex: `api/processa_ocorrencia.php`).
3. O **CONTROLLER** valida os dados e interage com o **MODEL** (ex: executa uma query via `conexao.php`).
4. O **CONTROLLER** processa o resultado e redireciona o usuário de volta para uma **VIEW** (ex: `dashboard.php`).
